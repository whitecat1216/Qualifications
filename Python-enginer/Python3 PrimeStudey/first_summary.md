## 1. Pythonの特徴・インタープリタ・コメント

### 問1：Pythonの特徴で**誤っている**ものはどれか

**答え：**  
「**ただ変数や引数の宣言は必要である。**」が誤り。

**サルでもわかる解説：**  
Python では **変数や引数を「型」で宣言する必要はありません**。  
`x = 10` と書くだけで整数、`name = "太郎"` と書けば文字列になります。  
「宣言は必要である」が誤りで、正しくは「**宣言は不要**」です。

---

### 問2：Pythonインタープリタで**誤っている**ものはどれか

**答え：**  
「**セカンダリプロンプトの記号はデフォルトでは「<<<」である。**」が誤り。

**サルでもわかる解説：**  
- **プライマリプロンプト** = 1行目で命令を待っているとき → 「`>>>`」  
- **セカンダリプロンプト** = 続きの行を待っているとき（for のあとなど）→ デフォルトは「**`...`**」（3つのドット）。  
「<<<」はセカンダリプロンプトではないので誤りです。

---

### 問3：コメントとして解釈され**ない**ものはどれか（# が含まれる行）

**答え：** **【E】**

**サルでもわかる解説：**  
`#` は「その行の # **以降**をコメントとして無視する」ルールです。  
**【E】** の `#あいさつ文はここまでです。` は **文字列の中**（クォートで囲まれた中）にあります。  
文字列の中の `#` は「ただの文字」なので、**コメントとしては解釈されません**。  
【A】～【D】はコードとして解釈される部分の # 以降なのでコメント扱いです。

## 2. 演算・文字列・スライス

### 問4：演算の実行結果

```python
a = 10
b = a ** 2    # 100
c = b % 20 + 5  # 0+5 = 5
d = 8
e = d / b     # 8/100 = 0.08
f = d // c    # 8//5 = 1
print('{0}, {1}'.format(e, f))
```

**答え：** **0.08, 1**

**サルでもわかる解説：**  
- `**` = べき乗、`%` = 余り、`/` = 割り算（小数）、`//` = 商（整数）。  
- e = 0.08、f = 1 なので、表示は **0.08, 1** です。

---

### 問5：文字列の連結とエスケープの実行結果

**答え：** **She said,"Heyyy! How are you?"**

**サルでもわかる解説：**  
- `\"` は「"」の1文字（エスケープ）。  
- `3 * "y"` は `"yyy"`。  
- `a = "She said,\"He" + "yyy" + "!"` → `She said,"Heyyy!"`  
- `print(a, b)` で間にスペースが入り、**She said,"Heyyy! How are you?"** になります。

---

### 問6：スライスとインデックスの実行結果（Zen = 'NowIsBetterThanNever'）

**答え：** **sNow**

**サルでもわかる解説：**  
`Zen[5]`, `Zen[10]`, `Zen[-7]`, `Zen[-3:-1]` の4つを連結すると **sNow** になります。  
インデックスは0始まりで、負のインデックスは後ろから数えます。  
スライスとインデックスで「s」「N」「o」「w」の4文字を取っていると押さえておけばOKです。

---

### 問7：エラーに**なる**操作はどれか（Zen = 文字列）

**答え：** **Zen[0] = 'J'**

**サルでもわかる解説：**  
文字列（str）は **イミュータブル（変更不可）** です。  
「1文字目を別の文字に置き換える」ような **Zen[0] = 'J'** は実行時エラーになります。  
スライスや `Zen[:1000] + 'J'` は新しい文字列を作るだけなのでエラーになりません。

## 3. 制御構文・ループ

### 問8：while とフィボナッチ（b < 10 の間だけ表示）

**答え：** **1,1,2,3,5,8,**

**サルでもわかる解説：**  
最初 a=0, b=1 → 表示 1。次 a=1, b=1 → 1。その次 a=1, b=2 → 2 … とフィボナッチ数列。  
b が 10 未満の間だけなので、最後は 8 まで。**1,1,2,3,5,8,** が正解です。

---

### 問9：リストの append / insert と for（colors[2:]）

**答え：** **green 5, blue 4, yellow 6,**

**サルでもわかる解説：**  
- append('yellow') → 末尾に追加。insert(0,'purple') → 先頭に挿入。  
- この時点で `colors = ['purple', 'red', 'green', 'blue', 'yellow']`。  
- `colors[2:]` は「インデックス2から最後まで」なので **green, blue, yellow**。  
- それぞれ `print(color, len(color), end=', ')` なので **green 5, blue 4, yellow 6,** と出力されます。

---

### 問10：素数判定の【A】（for-else）

**答え：** **【A】else:**

**サルでもわかる解説：**  
Python では **for に else を付けられます**。  
「for が **break されずに**最後まで回ったときだけ」else ブロックが実行されます。  
ここでは、割り切れる数がなかった＝**素数**のときだけ「is a prime number」と出したいので、for の直後に **else:** を置く形が正解です。

---

### 問11：range(-10, -30, -5) の結果

**答え：** **-10, -15, -20, -25,**

**サルでもわかる解説：**  
`range(-10, -30, -5)` は「-10 から -30 の**手前**まで、-5 ずつ」。  
-10 → -15 → -20 → -25 で、**終了値 -30 は含めない**ので -25 で終わりです。

---

### 問12：enumerate で同じ結果になる書き方

**答え：** **for i, v in enumerate(Zen):**

**サルでもわかる解説：**  
**enumerate(リスト)** は「(0, 1番目の要素), (1, 2番目の要素), ...」のように **インデックスと要素のペア** を渡してくれます。  
だから `for i, v in enumerate(Zen):` のあと `print(i, v)` で「0 Now」「1 is」… と同じ結果になります。  
`count` や `calculate`、`number` は標準にはありません。

## 4. 関数・引数・PEP8

### 問13：デフォルト引数の評価タイミング（arg = i）

**答え：** **5**

**サルでもわかる解説：**  
**デフォルト引数の値は、関数「定義」が実行されたときに一度だけ評価されます。**  
このコードでは、def が実行される時点で i が 5 だったと解釈されるため、`f()` を呼ぶと **5** が表示されます。  
後から i を 8 や 9 に変えても、arg のデフォルト値は変わりません。

---

### 問14：可変デフォルト引数（リスト）の罠【A】の出力

**答え：** **【A】の行の出力は ([1, 4, 9], [1, 27, 64])**

**サルでもわかる解説：**  
デフォルト引数でリスト `squares=[]`, `cubes=[]` を使うと、**そのリストは「関数定義時に1回だけ」作られ、呼び出すたびに同じリストが使い回されます。**  
- culc(1) → [1], [1]  
- culc(2,3) → [1,4], [1,27]  
- culc(3,4) → [1,4,9], [1,27,64]  
なので【A】の print(culc(3,4)) の結果は **([1, 4, 9], [1, 27, 64])** です。  
可変オブジェクトをデフォルト引数にすると「蓄積」が起きるので注意です。

---

### 問15：キーワード引数で正しい呼び出し

**答え：** **location(state='Tokyo', city='chiyoda')**

**サルでもわかる解説：**  
- `country` はデフォルトが 'Japan' なので省略可能。  
- `city` と `state` はキーワードで渡せば順序は自由。  
  **location(state='Tokyo', city='chiyoda')** が正しいです。  
- キーワード引数の後に位置引数を書くのは文法エラー。存在しない仮引数（zipcode）はダメ。引数なしでは city, state が足りないのでダメ。

---

### 問16：可変長引数【A】*args と **kwargs

**答え：** **【A】*arguments, **keywords**

**サルでもわかる解説：**  
- 名前の後の **"Open: 9:30 am"** などは「余った位置引数」なので **`*arguments`** でタプルとして受け取る。  
- **bouquet="Sunflower"** のようなキーワード引数は **`**keywords`** で辞書として受け取る。  
- 順序は「**先に *args、後に **kwargs**」なので **\*arguments, \*\*keywords** が正解です。

---

### 問17：PEP8 で**正しい**ものはどれか

**答え：**  
「**演算子の周囲やカッコの内側にもスペースを入れ読みやすさに配慮すべきである。**」

**サルでもわかる解説：**  
PEP8 では、読みやすさのために **演算子の前後やカンマの後などにスペースを入れる** ことが推奨されています（例：`a = 1`、`func(a, b)`）。  
- docstring は「冗長に」より「簡潔に」が推奨されることが多い。  
- コメントは「同じ行に限る」わけではない。  
- 国際化では UTF-8 が推奨され、ASCII が「常に最良」とは限りません。

## 5. リスト・データ構造

### 問18：リスト内包で [0, 1, 8, 27, 64] を作る

**答え：** **cubes = [x ** 3 for x in range(5)]**

**サルでもわかる解説：**  
リスト内包表記は **「[ 式 for 変数 in 反復可能オブジェクト ]」** の形です。  
「x を 0～4 で回して、各 x の3乗をリストにする」なので、  
**cubes = [x ** 3 for x in range(5)]** が正解です。

---

### 問19：二次元リスト【A】で power = [row[2] for row in matrix] → [5, 25, 125]

**答え：** **【A】[[2, 4, 8], [3, 9, 27], [5, 25, 125]]**（問題の正解として記載されているもの）

**サルでもわかる解説：**  
`power = [row[2] for row in matrix]` は「各 row の **3番目（インデックス2）** を取り出す」ので、  
matrix の各行の3列目が 5, 25, 125 になるような形です。  
※ 選択肢の [[2, 4, 8], [3, 9, 27], [5, 25, 125]] だと row[2] は 8, 27, 125 になりますが、試験の正解としてこの形が示されている場合は、**リスト内包で row[2]＝各行の3列目** という点を押さえておきましょう。

---

### 問20：deque の【A】と【B】

**答え：** **【A】collections　【B】queue.popleft()**

**サルでもわかる解説：**  
- **deque** は **collections** モジュールにあるので、【A】は **collections**。  
- 先頭の "bear" を除きたいので、**左から取り除く queue.popleft()** が【B】。  
- append で右に "goat" を足したあと、popleft() で先頭（bear）を削除し、pop() で右端（goat）を削除すると、['cow', 'dog', 'elephant', 'fox'] が残ります。

---

### 問21：リストの insert / append / sort / pop の結果

**答え：** **[50, 30, 20, 15, 5, 1]**

**サルでもわかる解説：**  
- 初期: [-10, 1, 15, 20, 30]  
- insert(2, 5) → [-10, 1, **5**, 15, 20, 30]  
- append(50) → [-10, 1, 5, 15, 20, 30, **50**]  
- sort(reverse=True) → [**50, 30, 20, 15, 5, 1, -10**]  
- pop(-1) → 末尾の -10 を削除 → **[50, 30, 20, 15, 5, 1]**

---

### 問22：スライス Zen[1:20:3]（Zen = 'ExplicitIsBetterThanImplicit'）

**答え：** **EpiiIB**

**サルでもわかる解説：**  
**Zen[1:20:3]** は「インデックス1から19まで、**3ステップごと**」に取ります。  
1, 4, 7, 10, 13, 16 番目の文字を並べると **EpiiIB** になります。

---

### 問23：データ構造で**誤っている**ものはどれか

**答え：**  
「**ディクショナリは変更不能（immutable）であるがキーの型は変更可能（mutable）であり……**」が誤り。

**サルでもわかる解説：**  
**ディクショナリ自体はミュータブル（変更可能）** です。要素の追加・削除ができます。  
一方、**キーに使えるのはイミュータブルなもの**（数値、文字列、タプルなど）に限られます。  
「ディクショナリは変更不能」という部分が誤りです。

## 6. 比較・モジュール・format

### 問24：対話モードで True が返るもの

**答え：** **'PHP' < 'Perl' < 'Python'**

**サルでもわかる解説：**  
文字列は **辞書順** で比較されます。  
PHP < Perl < Python は辞書順で成立するので、この式は **True** になります。

---

### 問25：モジュールのインポート検索順で**誤っている**ものはどれか

**答え：**  
「**あるモジュールがインポートされるときに……最後にシンボリックリンクを置いてあるディレクトリである。**」の部分が誤り。

**サルでもわかる解説：**  
正しい検索順序は **①ビルトインモジュール ② sys.path のディレクトリ** です。  
「最後にシンボリックリンクのディレクトリ」という特別な順序の説明は誤りです。

---

### 問26：モジュールの名前を確認する【A】

**答え：** **dir(sys)**

**サルでもわかる解説：**  
**dir(オブジェクト)** で、そのオブジェクトが持っている名前の一覧が得られます。  
sys モジュールの名前を確認するには **dir(sys)** が正解です。

---

### 問27：format の実行結果（math.pi, math.e）

**答え：** **2.718, 3.14159**

**サルでもわかる解説：**  
- **{1:.3f}** = 1番目の引数（math.e）を小数3桁 → **2.718**  
- **{0:.5f}** = 0番目の引数（math.pi）を小数5桁 → **3.14159**

## 7. 例外・ファイル・クラス

### 問28：例外処理（整数 a=5, b=0 を入力）

**答え：** **25, D, G**

**サルでもわかる解説：**  
25 が表示されたあと、0 除算で **ZeroDivisionError** が発生し **D** が表示されます。  
**else** は例外が起きなかったときだけ実行されるので今回は実行されません。  
**finally** は必ず実行されるので **G** が表示されます。

---

### 問29：エラーと例外で**誤っている**もの（パーサの矢印）

**答え：**  
「**エラーは矢印より後のトークンが原因である。**」が誤り。

**サルでもわかる解説：**  
構文エラーでは、**原因は矢印の位置そのものか、その前** にあります。  
「矢印より**後**が原因」という説明が誤りです。

---

### 問30：with 文のクリーンアップ

**答え：** **f.close()**（with を出るときにファイルが閉じられる）

**サルでもわかる解説：**  
**with** 文では、ブロックを抜けるときに **ファイルが閉じられる** 処理（f.close() に相当）が自動で行われます。

---

### 問31：例外の raise【A】【B】（OurException）

**答え：** **【A】OurException　【B】Exception**

**サルでもわかる解説：**  
【A】で OurException を発生させ、except 内で「intelligent」を表示したあと、【B】で外側の except に渡すために **Exception** を raise する、という流れです。

---

### 問32：クラスで**誤っている**ものはどれか

**答え：**  
「**__init__()メソッドに引数を与えることはできない。**」が誤り。

**サルでもわかる解説：**  
**__init__** には **引数を与えることができます**。  
通常は `def __init__(self, 引数1, 引数2):` のように定義し、インスタンス生成時に渡します。

---

### 問33：クラスのメソッド呼び出しとオーバーライド【A】【B】

**答え：** **【A】self.m()　【B】def d(self):**

**サルでもわかる解説：**  
【A】では同じクラスの m を呼ぶので **self.m()**。  
【B】では onimaru クラスに d メソッドを定義するので **def d(self):** が正解です。

## 8. その他

### 問34：sys.argv で ['one', 'two', 'three'] を得る

**答え：** **sys.argv[1:4]**

**サルでもわかる解説：**  
コマンドライン引数は **sys.argv** に入ります。  
[1:4] でインデックス 1, 2, 3 → **'one', 'two', 'three'** の3つが取れます。  
※ [0] はスクリプト名なので、引数だけ欲しいときは 1 から。

---

### 問35：正規表現の search でエラーになるもの

**答え：** **ret = prog.search('Kusaneiro')**

**サルでもわかる解説：**  
**search** はマッチしなければ **None** を返します。  
'Kusaneiro' はパターンにマッチしないので ret が None になり、**ret[0]** で TypeError になります。

---

### 問36：statistics モジュール（平均・中央値・分散）

**答え：** **mean, median, variance**

**サルでもわかる解説：**  
- **平均** → statistics.**mean**  
- **中央値** → statistics.**median**  
- **分散** → statistics.**variance**

---

### 問37：今日の日付を得る【A】

**答え：** **from datetime import date**

**サルでもわかる解説：**  
**date** クラスは **datetime** モジュールの中にあります。  
`date.today()` を使うには **from datetime import date** が正解です。

---

### 問38：logging の優先度（左から**高い**順）

**答え：** **CRITICAL、ERROR、WARNING、INFO、DEBUG**

**サルでもわかる解説：**  
重要度の高い順に **CRITICAL > ERROR > WARNING > INFO > DEBUG** です。  
※ second では「左から低い順」だったので、問題の言い方（高い順／低い順）に注意しましょう。

---

### 問39：pip uninstall で**誤っている**ものはどれか

**答え：**  
「**pip uninstall で削除対象となるパッケージの複数指定はできない。**」が誤り。

**サルでもわかる解説：**  
**pip uninstall** では **複数のパッケージ名を並べて指定できます**。  
例：`pip uninstall package1 package2`

---

### 問40：対話モードの補完で**誤っている**ものはどれか

**答え：**  
「**変数とモジュールの補完はインタープリタの起動時に自動で有効になっている。**」が誤り。

**サルでもわかる解説：**  
標準の対話モードでは、補完が「自動で有効」とまでは言えず、環境や readline の有無に依存します。  
そのため「起動時に自動で有効」という記述は誤りとされます。
